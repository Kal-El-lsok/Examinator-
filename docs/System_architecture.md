
# Examinator – System Architecture

## Data Flow

1. **Student enters exam hall**
   → WiFi camera captures face
   → AI matches against database
   → Verified/Unknown/Re-entry logged

2. **Exam begins**
   → 5 detection layers activate
   → AI monitors every desk in real-time

3. **Cheating detected**
   → System identifies within 2-5 seconds
   → Pinpoints exact seat location
   → Determines cheating type

4. **Automated response**
   → Audio warning plays
   → Dashboard alert flashes red
   → Invigilator notified

5. **Evidence saved**
   → Video clip saved
   → Audio recording saved
   → Screenshot captured
   → Database entry created
   → Ready for disciplinary action
